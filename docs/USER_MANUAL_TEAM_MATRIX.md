# 團隊與權限管理 - 資源矩陣操作手冊

本手冊說明如何在「團隊與權限管理」頁面中使用新的「資源矩陣」功能。

## 1. 介面概覽

進入「團隊與權限管理」頁面後，您會看到右上角有兩個切換標籤：
- **成員列表 (Member View)**：以人為核心，查看每個人管理了哪些帳號。
- **資源矩陣 (Resource Matrix)**：以帳號為核心，查看所有帳號的狀態與歸屬。

點擊「資源矩陣」即可進入矩陣管理模式。

## 2. 功能說明

### 2.1 檢視與篩選
- **搜尋**：在上方搜尋框輸入帳號名稱、ID 或平台名稱，可即時過濾列表。
- **狀態篩選**：透過下拉選單篩選「活躍」、「封禁」或「需驗證」的帳號。
- **平台篩選**：可單獨查看 Instagram、TikTok 或 Threads 的帳號。

### 2.2 新增/編輯帳號
- **新增**：點擊右上角「新增帳號資源」按鈕，填寫平台、名稱、ID 與狀態即可。
- **編輯**：點擊列表右側的「編輯」按鈕，可修改帳號資訊或重新指派歸屬。

### 2.3 快速指派 (單一)
在列表中「指派員工」欄位，直接點擊下拉選單，選擇新的負責人。系統會自動儲存變更，無需額外點擊確認。

### 2.4 批量指派 (Batch Assignment)
當您需要將大量帳號移交給某位員工（例如新員工入職，或老員工離職交接）時：
1. **勾選帳號**：點擊列表最左側的核取方塊（可多選，或點擊表頭全選）。
2. **出現工具列**：勾選後，上方會出現「批量操作工具列」。
3. **選擇對象**：在工具列的下拉選單中選擇目標員工。
4. **確認執行**：系統將自動將所有選取的帳號指派給該員工。

### 2.5 狀態監控
列表上方會顯示簡易統計數據：
- **活躍** (綠色盾牌)：正常運作的帳號數。
- **封禁** (紅色盾牌)：需要緊急處理的帳號數。

## 3. 常見問題 (FAQ)

**Q: 為什麼我看不到某些員工？**
A: 下拉選單僅顯示系統中角色為 `staff` 或 `team_leader` 的成員。

**Q: 刪除帳號後，原本的貼文會不見嗎？**
A: 系統僅刪除帳號的連結紀錄，不會刪除該帳號在社群平台上的實際貼文，也不會刪除系統內已生成的歷史文案紀錄（Assets 表）。

**Q: 可以將帳號指派給多個人嗎？**
A: 目前系統設計為「單一歸屬制」，一個帳號同一時間只能由一位員工負責。若需共同管理，請建立共用帳號或由組長進行協調。
